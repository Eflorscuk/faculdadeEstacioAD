Foi formalizada em 1991 através das propostas de Tim Berners-Lee. 2. Significa Hypertext Markup Language 3. Passou a ser mantida pela W3C em 1996 e HTML 4.01 publicada em 1999 4. Em 2000, a W3C recomenda a nova expecificação XHTML 5. Em 2008 surge HTML5, oficializada em 2014, sua principal característica é um novo mecanismo de extensibilidade que permite combinarsintaxes baseadas em XML dentro de uma mesma página HTML 5 é baseada em tags que devem controlar diversos aspectos do texto. No HTML 5 a preocupação é apenas estrutural, sendo delegado ao CSS o controle de características tipográficas A primeira linha do HTML define o tipo de documento !DOCTYPE html Depois definimos o cabeçalho e o corpo No cabeçalho encontram-se as informações de pré-carga, enquanto no corpo será colocada a parte visual tags comuns title Define o título da página, normalmente apresentado no topo da janela meta Permite a inclusão de informações e controle de características do documento, como o uso de UTF-8 para acentuação da língua portuguesa link Efetua a ligação da página com um recurso externo, como folha de estilo CSS ou uma biblioteca JS style Define uma folha de estilo interna para a página Algumas tags estruturais do body h1... Definem elementos de texto utilizados como títulos em diferentes tamanhos de fonte, sendo h1 para maior e h7 para o menos b, i, u, em, strong Negrito, itálico, etc... São consideradas obsoletas, sendo recomendado o uso de folhas de estilo CSS p, div Definem trechos de texto, sendo que p ressalta o parágrafo e div é muito utilizado para a criação de camadas ul, ol Iniciam a criação de listas de elementos, sendo numeradas com ol li Define um item de lista hr/ Quebra de linha table, tr, td Utilizadas para a cria'ão de tabelas. Hoje em dia seu uso é recomendado apenas para dados tabulares span Define um trecho avulso, ao qual podem ser aplicadas formatações CSS 1. CSS Cascade Style Sheets, a folha de estilo 2. Necessita o retorno booleano, indicando se os valores podem ser enviados ou não onclick Normalmente uma chamada explícita da validação. Bastante usado em botões de radio e caixas de marcação onchange Ocorre quando o valor (value) sofre uma alteração onfocus Ocorre quando o componente ganha o foco.
